# Design: Refactor UI 3 trang theo mockup mới (Chat / Friends / Contacts)

Generated by /office-hours on 2026-05-08
Status: AWAITING_CONFIRMATION
Mode: Builder

## Problem Statement

Có 3 mockup HTML trong `frontend/public/` (chat-smax-v3.html, friends-v2.html, contacts-approach-b-v3.html) thể hiện giao diện mới mong muốn. Code Vue 3 + Vuetify 4 hiện tại đang chạy với binding API thật, nhưng UI chưa khớp mockup. Cần refactor UI 3 trang đồng bộ với mockup mà không phá chức năng đang chạy, đồng thời chuẩn hoá top nav theo cấu trúc menu định nghĩa trong file Excel `Giao Diện Tính Năng Zalo Tool CRM.xlsx`.

## Constraints

- Stack: Vue 3 + Vuetify 4 + Pinia + Tiptap + Socket.io (đã cố định, không đổi).
- Backend hiện chưa có một số field mới mà mockup show (winner-nick, automation cards F1/F2/F3, AI suggest, label Zalo native, in/out count per-nick…).
- Mobile đã có `MobileLayout.vue` + `MobileChatView.vue` riêng — refactor chỉ áp cho desktop view, mobile để tách phase sau.
- Project có sẵn `DefaultLayout.vue` (sidebar nav + app bar light), GlobalSearch, NotificationBell.

## Premises (xác nhận TRƯỚC khi code)

1. **Top nav 9 tab vừa đủ.** Excel có Dashboard / Tin nhắn / Bạn bè / Khách hàng / Lịch hẹn / Phân tích / Báo cáo / Automation / Cài đặt. Automation + Cài đặt mở dropdown submenu khi click. Mockup chat-smax-v3 chỉ show 6 tab cứng — sẽ thay bằng 9 tab Excel.
2. **Workspace selector "HS"** ở top nav: phase 1 coi như single-tenant decoration (hiển thị tên workspace hiện tại). Multi-tenant switching để phase sau.
3. **MobileLayout giữ nguyên.** Refactor không đụng vào `MobileLayout.vue`, `MobileChatView.vue`, `MobileContactView.vue`. Top nav mới chỉ áp cho desktop (>=1024px).
4. **Field thiếu backend → ẩn động.** Không tạo placeholder cứng. Component nhận props, nếu null/undefined thì không render. Khi backend bổ sung sẽ tự động hiện.
5. **Design tokens tách file riêng.** `src/style.css` hoặc `src/assets/tokens.css` chứa CSS variables: `--primary: #2962ff`, `--primary-soft: #e3f2fd`, `--success: #00c853`, `--warning: #ff9100`, `--info: #2196f3`, `--error: #ff3d00`, các grey scale, `--header-bg: #1f2330`, `--bubble-self: #d0e6ff`, `--group-bg: #ffeae3`, `--female: #e91e63`, `--male: #1e88e5`. Vuetify theme đồng thời được cấu hình map sang các màu này để `<v-btn color="primary">` ra đúng `#2962ff`.
6. **Component đặc thù được trích xuất.** Filter rail, conv item, chip CRM (red/purple/orange/green/blue/yellow), label chip, gender badge, care-status pill, automation card, master/child accordion → thành Vue components dùng chung qua `src/components/ui/`.

## Cách tiếp cận đã chọn

### Phase 1: Shell + design tokens (1–1.5 ngày)
- File chạm: `src/layouts/DefaultLayout.vue` (rewrite), `src/plugins/vuetify.ts` (theme), `src/assets/tokens.css` (mới), `src/components/ui/` (mới — chứa shared chips, badges, avatars).
- Top nav 52px dark: logo + workspace selector + 9 nav tabs (Excel) + spacer + 4 icon-btn (Cài đặt, Đội nhóm, Tích hợp, Thông báo) + Avatar.
- Bỏ `<v-navigation-drawer>`. `<v-main>` full-width.
- Excel-driven menu: hardcode 9 tab cấp 1 + dropdown submenu cho Automation (Nhắn tin / Kết bạn / Bám đuổi) và Cài đặt (Tài khoản Zalo / API & Webhook / Tích hợp / Nhân viên / Phân quyền). Submenu Bám đuổi sâu hơn — giai đoạn này chỉ link đến `/automation/follow` cấp 2, route con để sau.
- `GlobalSearch` chuyển vào search box trong workspace selector hoặc icon-btn 🔍.
- **Done khi**: All 13 routes hiện tại (Dashboard, Chat, Groups, Friends, Contacts, ZaloAccounts, Appointments, Reports, Analytics, Settings, ApiSettings, Integrations, Automation) vẫn vào được; nav highlight đúng route active; theme dark/light toggle vẫn chạy.

### Phase 2: ChatView (2–3 ngày)
- File chạm: `src/views/ChatView.vue`, `src/components/chat/ConversationList.vue`, `src/components/chat/ChatContactPanel.vue`, `src/components/chat/MessageThread.vue`, mới: `FilterRail.vue`, `AISuggestBar.vue`, `AutomationCardList.vue`.
- Layout grid 4 cột: 290px FilterRail + 380px ConversationList + 1fr MessageThread + 350px ChatContactPanel.
- FilterRail (col 1): nick-picker (gom inbox), filter sections (theo người phụ trách, loại hội thoại, đọc/trả lời với toggle, theo thuộc tính). Bind vào Pinia store filter hiện tại nếu có; thiếu thì tạo `useChatFilterStore`.
- ConversationList (col 2): search input, label chip bar (TTAVIO/EGD/Phiền/Ấm/Nóng…), conv-item với avatar+platform mark+unread dot+tag row.
- MessageThread (col 3): chat-header với care-status pill + meta row, messages bubble (self vs other), AI suggest bar (`v-if="aiSuggestions.length"`), input area với toolbar trên + textarea + send-btn.
- ChatContactPanel (col 4): avatar big + form sửa inline (Tên Zalo, Tên Alias, Ngày sinh, Giới tính, SĐT đa dòng, Email, Địa chỉ, Nghề), section trạng thái nick × KH, 3 section tag (CRM hệ thống / Label Zalo / Tag riêng nick × KH), automation cards (running/scheduled/paused/completed), section "3 nick khác cũng chăm".
- Component đặc thù viết CSS thuần được phép bên trong `<style scoped>` của component, không global.
- **Done khi**: Chat hoạt động đúng như hiện tại + UI mới khớp mockup ~90%; field nào backend chưa có thì ẩn qua v-if.

### Phase 3: ContactsView (1.5–2 ngày)

1. **Workspace selector "HS"**: data lấy từ đâu? Hiện chưa có store/API multi-tenant. Phase 1 hardcode "HS holding" hoặc lấy từ `authStore.user.organizationName`?
2. **9 nav tab**: thứ tự hiển thị (Excel có ghi cấp 1 nhưng không rõ thứ tự ưu tiên). Đề xuất bám đúng thứ tự Excel: Dashboard / Tin nhắn / Bạn bè / Khách hàng / Lịch hẹn / Phân tích / Báo cáo / Automation / Cài đặt.

4. **Automation cards** trong ChatContactPanel: schema "F1/F2/F3" với step + trigger + state — backend có thực thể này chưa? Nếu chưa, ẩn section.
5. **Care status pill** ("⚡ Đang đàm phán giá"): có enum cố định không? Hay free-text? Mockup show 7 trạng thái: 💬 Quan tâm / 🤝 Chăm sóc / ⚡ Đàm phán / 🔥 Nóng / ❄ Lạnh / ✅ Đã chốt / 🆕 Mới.
6. **Label Zalo native** (per-nick): backend có sync label từ Zalo SDK chưa? Nếu chưa → ẩn section "Label Zalo (native)".
7. **Per-nick state KB** (Đã KB / Đã gửi mời / Đang nhắn lạ / Đã ngắt): cần kiểm tra schema friends table có column `friendship_status` per pair (nick × customer) chưa.
